## Executive summary

The Sigma codebase is, on the whole, in decent structural shape: the SQL layer is consistently parameterized, keyset pagination and the rollup-vs-aggregation split are sound, React/JSX auto-escaping neutralizes the untrusted registry text that flows into HTML, and the web app's CSP/nonce posture is well-constructed. The verification pass confirmed **one critical issue** that dominates the risk picture: the ETL worker ships to a public `*.workers.dev` URL with **no authentication**, and its `POST /etl/refresh` body is staged verbatim and promoted into the live public tables — meaning any internet caller can fabricate procurement records on an anti-corruption transparency site. Beyond that, the recurring theme is the **untrusted-feed contract being honored inconsistently**: the in-Worker OCDS port silently dropped the CLI loader's numeric/date validation, the daily `refresh-slice.sql` diverges from the full normalize on `annex_suspect` amounts (corrupting leaderboard sums), and all three CSV exports are open to spreadsheet formula injection. A second theme is **filter/scope drift in the UI**: the contracts FilterRail loses the authority/bidder scope, and the companies CSV exports the full unfiltered dataset under a filtered URL. The single most important risk to fix first is the unauthenticated ETL write path — it is the only finding that lets a remote attacker corrupt production data with no user interaction.

## Findings

### Critical

**Unauthenticated `POST /etl/refresh` injects attacker-controlled rows into the public DB** — `apps/etl/src/index.ts:108-114`. The ETL worker deploys to the default public `sigma-etl.<sub>.workers.dev` (no `routes`/`custom_domain`/auth in either wrangler config), and the request body is cast `as RefreshParams` and passed to `env.REFRESH.create()`. Caller-supplied `releases` skip the live АОП fetch, flow through `releaseToContracts → upsertContractStaging` into `raw_egov_contracts` as `ocds:fixture`, then `refresh-slice.sql` (`WHERE source LIKE 'ocds:%'`) promotes them into the live `authorities`/`bidders`/`tenders`/`contracts` tables the public site reads. **Impact:** any internet caller can fabricate authorities, bidders (arbitrary ЕИК/name), and contracts (arbitrary EUR amount) that appear as real data — defamation, fake corruption records, skewed leaderboards/risk scores. **Fix:** require a constant-time shared-secret/Access check on every non-`/health` route, disable `workers_dev`, and gate the `releases` fixture path behind a dev-only flag; validate the body shape instead of casting.

### High

**Toggling any rail filter on /contracts drops the authority/bidder scope** — `apps/web/app/routes/contracts.tsx:75-114` (FilterRail.tsx:54-97). The `<Form method="get">` emits only `sort` + the year/sector/procedure/value/eu group inputs; `authority`/`bidder` are not form fields, so any rail toggle submits without them and silently widens a single-authority/company drill-down to the whole dataset — while the "Филтрирано по…" banner still claims the scope is active. Sort/CSV links avoid this because they clone the full query via `withParams`. **Fix:** render hidden `authority`/`bidder` inputs in the Form, or seed it from the full `URLSearchParams`.

**`pageCursors` inverts Prev/Next for backward ('before') pages** — `packages/db/src/queries/keyset.ts:85-98`. For a 'before' page the list fetches ASC then reverses, so `hasMore` reflects the *previous* edge. `pageCursors` is direction-blind: it always assigns `hasMore` to `nextCursor` and emits `prevCursor` whenever the cursor decodes. Paging back to the first batch makes the Next button vanish (user stranded mid-dataset) and exposes a Prev link to an empty page. Only `authorities.tsx` papers over the empty-Prev symptom (via the unreliable display-only `page=N`); the disappearing-Next half is unmitigated in all three routes. *(Flagged by db-query-layer and web-list-routes.)* **Fix:** make `pageCursors` direction-aware — for a 'before' page always emit `nextCursor`, emit `prevCursor` only when `hasMore`; add a 'before'-cursor unit test.

**Companies CSV ignores sector/year/EU filters** — `packages/db/src/queries/companies.ts:162-193` and `apps/web/app/routes/companies.csv.tsx:6-13`. `streamCompaniesCsv` builds only `entityWhere(p)` and queries `FROM company_totals`, never calling `source(p)`; the CSV loader also forwards only sort/kinds/countBucket. So a sector/year/EU-filtered companies list exports the full unfiltered rollup, even though the on-page link forwards the filters. `streamAuthoritiesCsv` does call `source(p)`, proving the asymmetry is unintended. *(Flagged by db-query-layer and sec-export-untrusted — two layers of the same drop.)* **Fix:** parse/forward `sectors/years/eu` in the loader and stream `FROM ${source(p).from}`; add a test asserting CSV row set == list row set.

**CSV formula injection across all three exports** — `packages/db/src/queries/contracts.ts:299-303,338-357`; `companies.ts:178-182,200-214`; `authorities.ts:156-160,182-185`. The identical `csvCell`/`cell` helpers only quote on `[",\n]` and never neutralize leading `= + - @ TAB CR`. Untrusted registry names/subjects/settlements (e.g. `=HYPERLINK("http://evil",…)`) are emitted verbatim at cell start and evaluate as formulas in Excel/Sheets/LibreOffice — enabling HYPERLINK/WEBSERVICE exfiltration, content spoofing, and (with click-through) DDE. The missing `\r` in the trigger regex also lets a lone CR split rows. *(Two unit reviews and three call sites; merge into one fix.)* **Fix:** in a single shared helper, prefix-guard any value starting with `= + - @ \t \r` with `'` and force quoting; add `\r` to the quote-trigger regex; unit-test formula-prefixed and CR inputs.

**Worker OCDS port dropped the CLI loader's numeric validation** — `packages/ingest/src/ocds.ts:142-171`. The header claims parity with `load-ocds.mjs`, but that loader guards every numeric column via `Number.isFinite(Number(v)) ? … : 'NULL'`. The port passes `signing_value`, `estimated_value`, `bids_received` through verbatim into REAL/INTEGER D1 columns. From the UNTRUSTED feed, a non-numeric `amount` (e.g. `"5 000"`, `{}`, Infinity) is bound as-is: an object/Infinity throws and fails the whole batch ingest; a non-canonical numeric string silently coerces to 0.0 in downstream `value_flag`/`amount_eur` math. **Fix:** reintroduce a `Number.isFinite` coercion helper wrapping the numeric fields (and currency whitelist) before staging.

**`annex_suspect` amount_eur diverges between full normalize and daily refresh** — `scripts/refresh-slice.sql:111-116`. `normalize-egov.sql` falls `annex_suspect` rows back to `signing_value` for both `amount` and `amount_eur` (documented policy: the suspect annex is the bad part). `refresh-slice.sql` special-cases only `value_suspect` (→NULL) and uses `COALESCE(current_value, signing_value)` for `annex_suspect`, i.e. the inflated (≥100×) current value. After a daily refresh those rows feed `company_totals.won_eur`, `authority_totals.spent_eur`, `flow_pairs`, `home_totals` — silently wrong leaderboard/home figures until the next full normalize. **Fix:** mirror normalize's `trusted_native` CASE in refresh-slice; factor the `value_flag→amount` logic into one shared snippet so the two scripts cannot drift.

**Admin worker fails open when the Basic-auth secret is unset** — `apps/admin/src/index.ts:28-38`. `isAuthorized()` returns `true` unconditionally when `!env.ADMIN_BASIC_AUTH_PASS`, with no environment discriminator. The admin worker has a real `deploy` script but is **not** in the CI deploy workflow, so it's pushed by hand where the secret is easy to forget — exposing the DB-backed admin panel with zero auth. **Fix:** fail closed (return 503/401 in prod when no password is configured); allow the open path only behind an explicit dev flag, and make the deploy refuse to ship without the secret.

**Unauthenticated ETL trigger enables DoS / cost / cron collision** — `apps/etl/src/index.ts:108-119`. Independent of the `releases` vector, the same open `POST /etl/refresh` (+ `GET /etl/refresh/{id}`) lets anyone spin up unlimited `RefreshWorkflow` runs that fetch `data.egov.bg` and run a full derive-slice batch — burning Workflow/D1/subrequest quota, racing the 6-hourly cron, and amplifying load on the upstream registry; the status route leaks workflow state for guessed ids. **Fix:** apply the same auth gate as the critical finding to POST and GET; add rate-limiting and a concurrency guard.

### Medium

**INSERT OR IGNORE silently drops contracts hitting the amount NOT NULL constraint** — `scripts/normalize-egov.sql:202-215,287-290`. `contracts.amount` is `REAL NOT NULL`. An `annex_suspect` row with `signing_value` NULL and a negative `current_value` (classified via the `current_value < 0` branch) sets `amount = signing_value = NULL`, but the survival WHERE only checks `COALESCE(current_value, signing_value)`. `OR IGNORE` swallows the NOT NULL violation, dropping a real contract with no error/log — masked by the legitimate FK-orphan skips. **Fix:** `COALESCE` the `annex_suspect` amount (`COALESCE(signing_value, current_value)`) or tighten the WHERE; add a post-import count reconciliation.

**OCDS-refreshed contracts miss ~17 columns the full normalize populates** — `scripts/refresh-slice.sql:78-104`. The refresh INSERT lists 21 columns; normalize lists 36. Omitted (and NULL on `c:o:%` rows): `awarded_to_group`, `lot_id`, `document_number`, `eu_programme`, `duration_days`, `winner_size`, `contractor_country`, `bids_sme/rejected/non_eea`, subcontractor fields, `eauction`, `framework`, `accelerated`, `strategic`. Detail pages and lot-join logic (`getContract` reads these) silently behave differently for fresh vs. base contracts until a full normalize. **Fix:** bring the refresh INSERT to column parity (shared column-list/template).

**`refresh-slice` collapses distinct contracts sharing unp+contract_number via a bare GROUP BY** — `scripts/refresh-slice.sql:104-142`. The id is `'c:o:'||unp||':'||contract_number` and the statement ends `GROUP BY` that same expression while projecting many non-aggregated columns; SQLite picks an arbitrary row per group. When the feed has two contracts under one ocid with the same `contract_number` (different suppliers/values), all but one are dropped — under-counting awards and breaking the "deterministic re-run" claim. The full normalize keys by rowid and keeps every row. **Fix:** include supplier/lot in the id+GROUP BY, or use explicit aggregates / `ROW_NUMBER` for a stable collapse.

**Staging upsert is not atomic: DELETE commits before INSERT batches** — `packages/ingest/src/staging.ts:13-25`. The scoped DELETE runs as its own `.run()`, then inserts go out as multiple separate `db.batch()` calls (CHUNK=100). A mid-run failure after DELETE + chunk-1 leaves `raw_egov_contracts` partially wiped for that source. Workflow `step.do` retries re-run DELETE+INSERT idempotently (self-healing on success), and a thrown batch fails the step before derive-slice — so the realistic worst case is a temporarily truncated source, not live-domain corruption. **Fix:** fold DELETE + inserts into a single transactional `db.batch()` when row count permits, or make the DELETE the first statement of chunk 1.

**Cron 403 / egress freeze is silent** — `apps/etl/src/index.ts:42-69`. Discovery throws on any non-OK HTTP (`postJSON` on 403 — the documented production condition freezing data since 2026-05-24); `scheduled()` only creates the run and ignores the outcome. There's no last-success heartbeat, no alert, no 403-vs-"no new period" distinction. Worse, an empty-but-200 discovery still runs `derive-slice`, which advances `home_totals.refreshed_at` to now — surfacing apparent freshness while ingesting nothing. **Fix:** treat discovery errors / empty dataset sets as alertable (structured warning, heartbeat row, or freshness degradation), and don't advance `refreshed_at` on a zero-ingest run.

**Unauthenticated ETL trigger enables DoS / cost / cron collision** — `apps/etl/src/index.ts:108-119`. (See the High entry above; the verifier kept a medium-severity twin focused purely on the quota/cron-collision impact independent of the data-fabrication vector.) **Fix:** same auth gate + rate-limiting + concurrency guard.

**`splitSqlStatements` corrupts `--` inside string literals and merges same-line `;`** — `packages/ingest/src/refresh.ts:7-12`. The splitter strips `--[^\n]*` (removing `--` inside literals) and splits only on `;`+newline (merging `SELECT 1; SELECT 2;`). `refresh-slice.sql` survives today only by accident (no `--` in literals, one statement per line). The SQL is a build-time-bundled asset, so the trigger is a future authoring edit, not the untrusted data feed — hence medium, not high. **Fix:** literal-aware comment stripping and split on `;` before optional whitespace/comment then newline-or-EOF; better, keep statements as separate files/array. (Its test asserts robustness the function lacks — see Test coverage.)

**`published_at` regresses to NULL — port omits the `publishedDate` fallback** — `packages/ingest/src/ocds.ts:114`. The port uses `dateOnly(rel.date)`; the CLI loader uses `dateOnly(rel.date || meta.publishedDate)`. `publishedDate` is declared on `OcdsPackage` but never threaded into `OcdsMeta`, so releases lacking their own `date` (common — only the package carries it) now yield NULL, degrading date facets/sorting. **Fix:** thread `pkg.publishedDate` into `OcdsMeta` and use `rel.date ?? meta.publishedDate`.

**`detectPriceAnomaly` treats missing/zero reference price as parity** — `packages/analysis/src/price-anomaly.ts:18-24`. When `refPrice <= 0`, `deviationPct=0 → severity=0` ("fairly priced"), indistinguishable from a true parity price; `aggregatePriceSignal` averages these zeros in, diluting the real signal. No production caller yet, but it's a baked-in contract flaw in a shared API that will mask overpricing once wired in. **Fix:** return null/optional for `refPrice <= 0` and exclude unknowns from the aggregate denominator.

**Value-sort keyset wraps `amount_eur` in COALESCE, defeating its index** — `packages/db/src/queries/contracts.ts:25-30,164-169`. The value/date sorts use `COALESCE(c.amount_eur, …)` and the keyset predicate inlines that expression on both sides, so SQLite cannot use `idx_contracts_amount_eur`/`idx_contracts_signed` for the seek. Deep pages of the **default** value-sorted list (~190k rows) fall back to scan+sort — undercutting the module's "O(1) at any depth" claim and metering more D1 rows read. **Fix:** add an expression index matching the COALESCE form, or split the NULL partition with a separate ordering branch so the bare-column index applies; at minimum soften the header comment.

**Resource routes / CSV exports receive no security headers** — `apps/web/app/entry.server.tsx:52-56`. `securityHeaders()` (nosniff, X-Frame-Options, Referrer-Policy, CSP, HSTS) is applied only in the SSR HTML path. Loaders returning a `Response` directly (`contract.json`, `robots`, all sitemaps, all CSVs, streamed responses in `sitemaps.ts`) bypass it entirely; `contract.json` echoes untrusted registry fields as JSON. *(Verifiers down-rated to low — explicit Content-Types blunt sniffing on modern UAs; merged from sec-web-headers-xss and web-detail-routes.)* **Fix:** apply at least nosniff + X-Frame-Options + Referrer-Policy (HSTS in prod) to every response in the worker fetch handler `apps/web/workers/app.ts`, not only the SSR path.

## Round 2 — addendum (blind-spot sweep)

_Round 2 method: 5 targeted finders over the areas no Round-1 unit deeply opened (PII/GDPR public-leak trace, the standalone `apps/api` worker, the operator-run `scripts/*.mjs` CLI loaders, supply-chain/CI/deploy guards, and the edge-cache wrapper), same adversarial multi-skeptic verification. **Funnel:** 19 raw → 17 confirmed (2 refuted)._

**Round 2 severity:** 0 critical · **1 high** · 7 medium · 9 low. **By category:** security 8 · privacy 3 · correctness 3 · quality 3.

### Round 2 summary

Nothing here rivals the Round-1 critical (unauthenticated ETL write path), but two findings sit just below it. On PII: the `company_owners`/`beneficial_owners` tables are real but **never served**, so they are a data-minimization/retention gap rather than a live exposure — however, **`bidders.name` genuinely publishes natural-person (ЕТ / individual-winner) full names** to indexed, crawlable, bulk-CSV and `/contracts/:id.json` surfaces with zero redaction, which is the most serious confirmed item (and the direct legal hook for the GDPR/CJEU analysis in `compliance-*.md`). On the CLI side, **`load-fx.mjs` has a fully-reproduced SQL-injection breakout** (raw ECB rate interpolated unguarded → `DROP TABLE` executes against D1, including `--remote` production), bounded only by being operator-run. The `apps/api` worker is parked and clean on injection/limits but ships no security headers and mislabels `authorityName` with the raw `authority_id`.

### High

**`load-fx.mjs` interpolates the ECB rate into SQL with no numeric guard (reproduced injection + invalid-token breakage)** — `scripts/load-fx.mjs:52-53,61,67`. `rate = j?.rates?.EUR ?? null` is taken verbatim from frankfurter.app JSON and interpolated raw as `${r.rate}` into the VALUES tuple — the only field with no quoting and no `Number()/isFinite` gate (every other loader has one). The lone filter `if (rate == null) continue` lets any non-null value through. **Reproduced:** a `rates.EUR` of `0.51); DROP TABLE fx_rates;--` emits a statement that closes the INSERT and runs the DROP under sqlite3; `Infinity`/`NaN`/bare tokens abort the whole `--file` load. A compromised, MITM'd (plain `fetch`, no pinning), or merely format-changed FX endpoint injects arbitrary SQL against D1 — including `--remote` production (applied via `wrangler d1 execute … --file`). **Fix:** `const n = Number(rate); if (!Number.isFinite(n)) continue;` and interpolate `n`.

### Medium

- **`bidders.name` served verbatim with no natural-person (ЕТ) redaction** — `scripts/normalize-egov.sql:153-161`. `bidders.name = MIN(contractor_name)` verbatim; `kind` is only `company|consortium` (`EntityKind` has no natural-person value — `api-contract/src/index.ts:71`), so a Bulgarian ЕТ sole trader / individual winner's full personal name flows through unchanged to the company profile (`company.tsx`, no noindex), contract page, `search_index` (`precompute.sql:161,166`), the bulk companies CSV (`companies.ts:204`), the company sitemap (`sitemaps.ts:94-96`), and `/contracts/:id.json` via `sourceNames.bidder` (`details.ts:556`). **Impact:** a real person's name is published, indexed, crawlable, bulk-downloadable, JSON-exposed, and linkable to amounts/counterparties. **Fix:** classify natural persons at ingest — `bidders.legal_form` is already populated (`normalize-egov.sql:312` → `legalForm` at `details.ts:183`), so the hook exists — and suppress/pseudonymize on public surfaces; at minimum drop them from the CSV and `/contracts/:id.json sourceNames`. *(See the GDPR/AML/CJEU analysis in the compliance map.)*
- **Beneficial-owner / company-owner names persisted in production D1 but never served** — `scripts/normalize-egov.sql:319-329`. `company_owners`/`beneficial_owners` are rebuilt each run from `raw_tr_*`, carrying natural-person `owner_name` (`load-tr.mjs:135,151`; EGN not stored) as real columns (`0000_init.sql:169,182`). A repo-wide grep finds **zero** `SELECT/JOIN` against either table. **Impact:** PII at rest with no serving purpose, no documented lawful basis/retention/erasure — a GDPR storage-/purpose-limitation gap where one future query, export, or DB dump becomes mass disclosure. **Fix:** document lawful basis + retention; if parked, keep only `owner_eik` edges and drop the name columns (or store names off-production); else add an erasure job and access boundary.
- **`load-ocds.mjs` numeric coercion validates the parsed number but emits the original string** — `scripts/load-ocds.mjs:70-74`. `sqlLiteral` guards on `Number(value)` but emits the unmodified `String(value)` for monetary/bid columns. `"0x10"`/`"1e3"`/`" 5 "` pass and are emitted verbatim; `"0b101"`/`"0o17"` pass `Number.isFinite` but emit tokens sqlite3 rejects, breaking the whole INSERT batch. **Fix:** `const n = Number(value); return Number.isFinite(n) ? String(n) : 'NULL';`.
- **Feed-supplied `resourceUri` used unsanitized as a cache file path (path traversal on `writeFileSync`)** — `scripts/load-ocds.mjs:88,97-98`. `resolve(cacheDir, `${resourceUri}.json`)` with `resourceUri` the untrusted `uri` from `listResources`; `resolve()` does not contain it — reproduced `../../apps/api/wrangler` and `../../../etc/cron.d/x` escaping the repo. `load-tr.mjs:68` has the identical `.xml` pattern. **Fix:** sanitize to `[A-Za-z0-9_.-]` and assert the resolved path stays within `cacheDir`.
- **`load-tr.mjs` truncates its output `.sql` every chunk — chunked dry runs silently keep only the last chunk** — `scripts/load-tr.mjs:182,233-237,284`. Each iteration re-opens the three output files with `createWriteStream` (default `'w'`, truncates); rows persist only via the per-chunk D1 load gated on `if (apply)`. With `--apply` omitted (the documented dry-run that writes `data/tr-*-load.sql`), every chunk overwrites the previous — `--all` (chunk 40) drops all but the last ~40 files. **Fix:** open the streams once before the loop and append.
- **`xlsx@0.18.5` (npm build, two unfixed advisories) parses the untrusted АОП registry export** — `scripts/load-admin.mjs:24,315-321`. Pinned `^0.18.5` → npm build subject to GHSA-4r6h-8v6p-xvw6 (prototype pollution, fixed only in 0.19.3) and GHSA-5pgg-2g8v-p4x9 (ReDoS, fixed only in 0.20.2), neither fixable via npm. `XLSX.read` content-sniffs by first byte, so a tampered file beginning with `<` still routes to the vulnerable XML parser. **Fix:** the loader only needs `header:1` row arrays — drop SheetJS for a minimal RFC-4180 splitter, or switch to the patched CDN build.
- **Edge-cache wrapper replays full headers/body with no `Vary` or anonymous guard: frozen CSP nonce now, latent cross-user leak later** — `apps/web/workers/app.ts:40-58`. Caches GET responses keyed only on URL+deploy-tag and on a HIT replays stored body+headers verbatim; the per-request CSP nonce lands in both, so one nonce serves all visitors for 300–3600s. `apps/web` has zero `Set-Cookie`/session use today, so the cross-user *data* leak is latent, not present — but the cache is fail-open. **Fix:** use a nonce-less CSP (hashes/`strict-dynamic`) on cacheable routes, or rewrite CSP+nonce on every served response including HITs; add an anonymous-only guard before `put`.

### Low

- **`authorityName` is populated with the raw `authority_id` (`auth:<ЕИК>`), never resolved** — `apps/api/src/index.ts:27`. `toSummary()` sets `authorityName: t.authority_id` with no JOIN to `authorities`; every `/api/tenders[/:id]` response mislabels the contracting authority as an opaque prefixed id. **Fix:** LEFT JOIN `authorities` and select `a.name`.
- **Public JSON API worker sets no security response headers** — `apps/api/src/index.ts:15-20`. The `json()` helper sets only `content-type` — no nosniff/X-Frame-Options/Referrer-Policy/HSTS, though `apps/web/app/lib/security.ts` already provides them. **Fix:** factor an `apiSecurityHeaders()` and apply it in `json()`.
- **`Env` declares a `CACHE` KVNamespace binding that does not exist in `wrangler.toml`** — `apps/api/src/index.ts:10-13`. Latent footgun: the first `env.CACHE.get(...)` throws in production with no compile-time warning. **Fix:** remove `CACHE` until provisioned, or add the binding.
- **`load-fx.mjs` builds the frankfurter URL from a DB-derived currency without validation** — `scripts/load-fx.mjs:40-43,48,51`. `currency`/`contract_date` interpolated unencoded into the query string; bounded (fixed host, no SSRF). **Fix:** validate `^[A-Z]{3}$` / ISO date and `encodeURIComponent`.
- **Consortium-member name fragments from `bidders.name` emitted as `/search?q=name` links** — `apps/web/app/routes/company.tsx:118-129`. Commit `3465dbb` makes each unresolved participant a live search link; a fragment that is a natural-person name becomes a one-click cross-record search. Bounded (parser routes enumerations to plain prose, target is `noindex`). **Fix:** only auto-link fragments that resolve to a company.
- **Deploy guard omits `CLOUDFLARE_ACCOUNT_ID` — a required deploy credential** — `.github/workflows/deploy.yml:43-51`. The guard checks only `CLOUDFLARE_API_TOKEN`+`SIGMA_D1_ID`; with a multi-account token, wrangler may target an unintended account. **Fix:** add the third check.
- **CI runs no dependency audit, secret scan, or lint gate** — `.github/workflows/ci.yml:18-20`. Install/typecheck/test only — no `pnpm audit` (would flag the xlsx class), no secret scanner, `lint` never invoked, on an app handling production registry credentials. **Fix:** add `pnpm audit --audit-level=high`, gitleaks, and wire `pnpm lint`.
- **`packages/config` has zero tests for its deterministic tables and the risk-weight sum invariant** — `packages/config/src/index.ts:289-295,135-138`. `sectorForCpv`/`procedureGroup`/`requireEnv` and the `DEFAULT_RISK_WEIGHTS` sum-to-1.0 invariant (asserted only in a comment) are untested. **Fix:** add `index.test.ts` covering the weight sum, procedure mapping fallback, CPV extraction, and `requireEnv`.
- **Hand-rolled `sqlLit`/`sqlStr` escape only the single quote — no NUL/control-char normalization** — `scripts/load-tr.mjs:43` (+ `load-egov.mjs:128`, `load-annexes.mjs:99`, `load-admin.mjs:80`, `load-fx.mjs:28`). Sufficient against the wrangler SQL splitter (not injection), but embedded newlines/tabs/NUL ride into D1, FTS, and display. **Fix:** strip C0/NUL in the escapers before quoting.
